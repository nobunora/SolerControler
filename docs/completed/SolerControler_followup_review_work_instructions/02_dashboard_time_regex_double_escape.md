## 02. 外部JavaScript内の時刻正規表現が二重エスケープされている

### 優先度
**P0 / 最優先**

### 対象
- `static/dashboard.js`
- `estimateHourlyNightGridCharge()`
- 時刻解析用`minuteOf()`

### 問題箇所
`static/dashboard.js:1021` 相当:

```javascript
const match = /^(\\d{1,2}):(\\d{2})$/.exec(String(value || "").trim());
```

外部`.js`ファイル内の正規表現リテラルでは、数字クラスは `\d` と記述する。
現在の `\\d` は数字ではなく、バックスラッシュと文字`d`を要求する。

### 再現結果
Nodeで再現済み:

```text
02:43 => false
\d\d:\d\d => true
```

したがって実際の `02:43`、`04:00`、`07:00` などはすべて解析失敗する。

### 影響
解析失敗時は次のfallbackへ入る。

```javascript
start = 4 * 60;
end = 7 * 60;
```

そのため、実際の`charge_start_time`と`charge_end_time`を使って夜間充電量を時間配分する修正が機能せず、常に04:00〜07:00へ配分される。

### 根本原因
Python文字列内に埋め込まれていたJavaScriptを外部ファイルへ移動した際、Python用のエスケープが残った。

### 修正方針
次へ変更する。

```javascript
const match = /^(\d{1,2}):(\d{2})$/.exec(String(value || "").trim());
```

### 調査済み範囲
`static/dashboard.js` 内で `\\d`、`\\s`、`\\w` 等の正規表現二重エスケープ候補を検索した結果、この1箇所だけ確認された。

### 必須テスト
JavaScript単体テストを追加し、最低限次を確認する。

- `02:43` → 163分
- `04:00` → 240分
- `07:00` → 420分
- `23:30` → 1410分
- `24:00` → 無効
- `2:3` → 無効
- 空文字 → 無効

`estimateHourlyNightGridCharge()`について次も確認する。

- 02:30〜04:30を0時〜23時行へ正しく比例配分する。
- 23:30〜01:30の跨日区間を正しく配分する。
- 配分値合計が`night_charge_kwh`と一致する。
- 開始=終了の場合は0配分。
- 時刻欠損時だけ04:00〜07:00 fallbackを使う。

### 完了条件
- 実時刻文字列が正しく解析される。
- 実スケジュール区間に応じて充電量が配分される。
- 総充電量が保存される。
- ブラウザまたはNodeベースの自動テストで再発を検知できる。
