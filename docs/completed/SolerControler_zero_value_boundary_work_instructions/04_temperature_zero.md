## 04. 有効な予報気温 `0℃` を既定値 `20℃` に置換する

### 優先度

**P2**

### 対象

- `energy_model_main.py`
- 予報気温の正規化
- `temp_c`
- `app/energy_model.py`
- `forecast_pv_energy_kwh()`
- `effective_capacity_kwh()` または温度補正経路
- `tests/test_energy_model_runtime.py`
- `tests/test_energy_model.py`

### 調査結果

現行コードでは予報気温を次のように取得している。

```python
temp_c = _to_optional_float(forecast.get("temp_c")) or 20.0
```

Pythonでは `0.0` が偽として評価されるため、冬季に一般的な0℃が欠損値と同じ扱いになり、20℃へ置換される。

### 再現結果1: PV予測

同一係数、同一日照時間5時間で比較した。

```text
pv_at_0c_kwh = 10.875
pv_at_20c_kwh = 10.175
difference_kwh = 0.6999999999999993
```

標準的な温度係数 `-0.0035` では、0℃を20℃へ置換するとPV予測が約0.70 kWh変化した。

### 再現結果2: 夜間充電量

複数の到達可能な入力条件を探索し、0℃と20℃の差が最終必要夜間充電量へ与える影響を確認した。

最大差を確認した条件:

- 現在SOC: 5%
- 日照時間予測: 8時間
- 日中負荷: 20 kWh
- 朝負荷: 0 kWh
- PV上書きなし
- その他条件同一

結果:

```text
required_at_0c = 2.333333333333335
required_at_20c = 3.5777777777777766
difference_kwh = 1.2444444444444418
pv_at_0c = 17.4
pv_at_20c = 16.28
```

有効な0℃を20℃へ置換することで、必要夜間充電量が約1.24 kWh増えるケースを確認した。

### 根本原因

1. 数値の既定値処理に `or` を使用している。
2. `None` と0℃を区別していない。
3. 温度入力の境界テストがない。
4. PVアレイ予測が利用できる場合は温度誤差が隠れ、フォールバック時だけ顕在化するため検出しにくい。
5. 気温とバッテリー温度の責務が近接し、どの温度へ既定値が適用されたか分かりにくい。

### 影響

- 冬季のPV発電量予測が歪む。
- 夜間充電量が過大または過小になる。
- バッテリー実効容量の温度補正にも同じ値が使われる場合、誤差が重なる。
- PV予測0 kWhの欠損化により旧来モデルへ切り替わった場合、この不具合が同時に発生する。
- 予報値として0℃を保存・表示していても、内部計算では20℃になる可能性がある。
- 説明可能性が低下する。

### 値の契約

| 入力状態 | 意味 | 処理 |
|---|---|---|
| `None` | 気温欠損または解析失敗 | 既定値20℃または明示的フォールバック |
| `0.0` | 有効な0℃ | そのまま採用 |
| 負値 | 有効な氷点下気温 | そのまま採用 |
| 正値 | 有効な気温 | そのまま採用 |
| `NaN` / `inf` | 不正 | エラーまたは欠損扱い |

0℃だけでなく、負の気温も有効値として保持する。

### 修正方針

truthinessを使用せず、Noneだけを既定値へ置換する。

```python
parsed_temp_c = _to_optional_float(forecast.get("temp_c"))
temp_c = 20.0 if parsed_temp_c is None else parsed_temp_c
```

同型の温度既定値処理が他ファイルに存在する場合は検索する。ただし、このコミットでは予報気温の同一問題だけを修正する。

### 推奨改善

予報気温とバッテリー温度を別変数・別契約で扱う。

```python
weather_temp_c: float
battery_temp_c: float
```

バッテリー温度が欠損した場合に外気温を代用するなら、その代用理由を出力へ残す。

例:

- `battery_temp_source=measured`
- `battery_temp_source=weather_fallback`
- `battery_temp_source=default`

- 空文字列の契約
