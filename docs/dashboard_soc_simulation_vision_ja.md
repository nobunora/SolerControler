# ダッシュボード: SOC 1日シミュレーション表示構想

## 目的

SOC判断フローを制約付き候補評価型へ整理した後、ダッシュボード上で「なぜそのSOC設定になったか」を確認できる表示を追加する。

この構想は将来検討用であり、現時点では実装しない。

## 表示したい内容

- 1日シミュレーション結果
  - 時間別PV予測
  - 時間別負荷予測
  - SOC推移
  - 買電/売電/余剰
  - 夕方SOC 0リスク
- 候補SOCごとの評価結果
  - 安全リスク
  - 経済コスト
  - 売電機会損失
  - 昼間買電期待額
  - peak未達ペナルティ
  - 最小スコア候補
- 最終判断の説明
  - 採用SOC
  - KP-NET丸め後SOC
  - 充電開始/終了時刻
  - 効いた制約
  - 却下された主な候補
  - `plan_quality`
  - `decision_rationale`

## UI方針

- ダッシュボードの一番下に追加する。
- 初期表示ではコンパクトに閉じる。
- パラメータ、候補一覧、詳細なコスト内訳は常時折りたためるようにする。
- 通常運用では「最終SOCと理由」だけを素早く読めるようにする。
- 調査時だけ、候補SOCごとの詳細評価を開けるようにする。

## 検討タイミング

まず既存のSOC判断フローを整理し、以下が固まってから検討する。

- Hard constraint / Risk constraint / Objective の分類
- `plan_quality` の定義
- `decision_rationale` の出力形式
- 候補SOCごとの評価結果フォーマット
- 過去データでの評価方法

## 次の推奨ステップ

条件木整理が完了した後、設計レポートの出力形式をもとに、ダッシュボードへ渡すJSON構造を決める。
